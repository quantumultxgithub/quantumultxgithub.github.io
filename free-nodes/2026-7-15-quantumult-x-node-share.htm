工具组合的核心价值

小火箭：iOS端的网络自由之门

V2Ray：新一代代理架构的革命者

作为Shadowsocks的进阶方案，V2Ray采用模块化设计，具备以下优势：
- 协议伪装：可模拟正常HTTPS流量，有效对抗深度包检测（DPI）
- 多路复用：通过WebSocket、gRPC等协议提升连接稳定性
- 分流控制：支持按域名、IP、地理位置等规则智能路由流量

七步完成配置全流程

第一步：基础环境准备

  1. 外区Apple ID获取（建议美/日/港区）
  2. App Store搜索"Shadowrocket"完成购买安装
  3. 准备可用的V2Ray订阅链接（付费服务推荐或自建服务器）

第二步：配置导入的三种方式

方式一：订阅链接自动更新
点击应用右上角"+" → 选择"Subscribe" → 粘贴订阅URL → 设置自动更新周期（建议12小时）

方式二：手动输入参数
进入手动配置界面需填写：
- 服务器地址（支持域名/IPv4/IPv6）
- 端口（通常443或8443）
- 用户ID（UUID格式）
- 加密方式（推荐auto或chacha20-poly1305）
- 传输协议（WS/mKCP/gRPC等）

方式三：扫描二维码导入
许多服务商提供配置二维码，点击小火箭顶部扫描图标即可快速导入

第三步：协议深度配置指南

以最常用的WebSocket为例：
1. 路径(path)：通常设置为"/ray"或随机字符串
2. Host头：填写服务商提供的伪装域名
3. TLS设置：务必开启并选择"tls"作为安全类型
4. 混淆选项：建议开启TLS1.3支持

第四步：连接测试与诊断

成功连接后，建议进行三项验证：
1. 访问whoer.net检查IP和DNS泄漏
2. 使用Speedtest测速确认带宽利用率
3. 尝试访问Google/Youtube等网站验证可用性

FAQ

Matsuri 节点延迟排序功能如何使用？
Matsuri 会定期测速节点延迟，并结合策略组自动选择延迟最低节点。用户无需手动切换即可保持高速访问，优化网页、视频和游戏体验，提高网络稳定性。
Clash 如何防止规则文件冲突导致分流异常？
合理组织规则顺序，将更具体的规则放在前面，通配规则放在末尾；使用 rule-providers 分离不同来源的规则，避免重复条目，并在变更后使用日志或 debug 模式检验分流是否按预期生效。
Clash 策略组如何提升节点切换效率？
通过策略组可设置节点优先级和分流规则，结合延迟检测功能，Clash 自动选择延迟低、稳定性高的节点处理不同流量，提高网页、视频和游戏访问速度及整体网络体验。
WinXray 的节点导入方式有哪些？
WinXray 支持手动添加节点、从 URL 订阅导入节点，以及批量导入 JSON 配置文件。用户可以直接复制订阅地址或粘贴节点链接，程序会自动解析并添加到节点列表中，操作十分便捷。
SSR 的协议和混淆组合如何选择？
根据网络封锁情况选择合适协议和混淆组合，例如 tls1.2_ticket_auth 模拟 HTTPS 流量。选择最佳组合可提高抗封锁能力和稳定性，保证节点长期可用。
WinXray 多策略组如何管理大量节点？
多策略组可将节点按地区、延迟或用途分类，并结合规则文件自动分流流量。这样可快速选择最优节点，提高访问速度和稳定性，同时优化节点管理和维护效率。
Matsuri 的延迟测试准确吗？
Matsuri 的延迟测试使用 ICMP 与 TCP 双重检测方式，可较准确反映节点实际响应时间。不过受网络波动影响，建议多次测试并结合速度测试结果综合判断。
VMess AEAD 加密如何确保数据安全？
AEAD 加密防止数据篡改、重放攻击和流量特征泄露。在高封锁环境下，保证节点可用，提升数据安全和连接稳定性，降低被检测风险，确保长期安全访问。
Nekobox 可以与 Shadowsocks 等客户端同时使用吗？
可以。Nekobox 配置独立，与 Shadowsocks、Clash 等客户端互不干扰。用户可根据需要通过策略组选择节点，实现灵活的网络管理和分流策略。
VLESS 的伪装流量如何提升隐蔽性？
VLESS 通过 Reality 或 XTLS 模式对流量进行伪装，使其在外观上与正常 HTTPS 请求一致。这样可以绕过深度包检测和封锁，提高高封锁网络环境下的可用性。
